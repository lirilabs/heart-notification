      Sends to all users — runs auto at 12 PM IST
    

    


      
      

        
🔑 What is CRON_SECRET?

        CRON_SECRET is a secret password you create yourself to protect your cron endpoint
        so only authorised callers (Vercel's scheduler or this console) can trigger it.
        Without it, anyone who knows your URL can blast notifications to all users.
        


        How to set it up in 3 steps:
        
    
          
  1. 
            Generate a secret — run in your terminal:
    
            openssl rand -base64 32
    
            You'll get something like: Xk3pQ9mN2vR7yTwL...
          
    2. 
          
  2. 
            Add it to Vercel →
            vercel.com/dashboard
            → Your Project → Settings → Environment Variables →
            Add CRON_SECRET = (your generated value) → Save → Redeploy.
          
    3. 
          
  3. 
            Paste the same value below in the CRON_SECRET field.
            This console will send it as Authorization: Bearer <secret>.
          
    4. 
        

        

        If you have NOT set CRON_SECRET in Vercel, leave the field below blank —
        your backend skips the check entirely.
      


      

        Vercel fires this automatically at 06:30 UTC → 12:00 PM IST every day.

4Android Payload Fix

  

    

      BUG
      Android message is too big
      ✓ Fixed in v2
    

    

      

        
❌ Root Cause

        FCM has a 4 KB hard limit on the entire push payload including the data object.
        Your images are stored as base64 strings in Firestore — a single WebP/PNG can be
        100 KB+. When the backend did imageBase64: base64Image inside the FCM data field,
        the payload exploded past the limit and FCM rejected it with "Android message is too big".
      

      

        
✅ What the Fix Does (update your backend files)

        The fix is one line removed from your getPrompt() function in both
        send-to-user.js and morning-push.js:
        


        Remove this from the extra object:

        ...(base64Image ? { imageBase64: base64Image } : {})
        


        Base64 images must never go into FCM's data payload.
        Instead, your app should fetch the image from Firestore after the user taps the notification,
        using the promptId that is already sent as a data field.
        


        Updated flow:
        
    
          
  1. FCM sends lightweight notification: title, body, promptId, category, authorName
    2. 
          
  2. User taps notification → app opens
    3. 
          
  3. App uses promptId to fetch full prompt doc from Firestore (including images)
    4. 
          
  4. Images display inside the app — no FCM size limit applies
    5. 
        

      


      

        
// BEFORE — causes "too big" error

        
const extra = {

        
promptId, promptText, category, authorName, authorPhoto,

        
...(base64Image ? { imageBase64: base64Image } : {}),  ← REMOVE THIS

        
};

        

        
// AFTER — stays under 4 KB ✓

        
const extra = {

        
promptId, promptText, category, authorName, authorPhoto,

        
};
